⚙️ Architecture

🔑 Replication & Consistency

  • • Replication factor: 2 (read quorum=2)
  • • Consistency mode: consistent — prevents restic AEAD corruption
  • • Layout version v209 — auto-applied via GarageCluster CR

📥 common-ingress (Tailscale VPN Inbound)

  • • 3 replicas per cluster — Tailscale VPN traffic → service
  • • For Tailscale VPN clients accessing Garage S3/admin/web
  • • Garage storage + gateway pods accept VPN inbound via this group
  • • Not used for public S3 CDN traffic (that goes through Envoy Gateway)
  • • 8–9 TS services per cluster with annotation: proxy-group: common-ingress
  • • Migrated from standalone TS pods to proxy group (Jun 9)

📤 common-egress (Outbound RPC)

  • • 3 replicas per cluster — outbound Tailscale connections
  • • Each cluster runs all 3 regions' gateway RPC services (13 total)
  • • Template: {zone}-gw-{ordinal}.keiretsu.ts.net:3901
  • • Cross-region RPC flow: Garage → common-egress → VPN → common-ingress → peer

🌐 GSLB + Envoy Gateway — s3.cdn.keiretsu.top

  • • K8GB round-robin DNS: s3.cdn.keiretsu.top → 3 A records
  • • Weights: ottawa 33%, robbinsdale 33%, stpetersburg 34%
  • • GSLB → Envoy Gateway public → garage-gateway:3900
  • • Tailscale VPN clients hit common-ingress proxy for S3 access
  • • All 3 clusters healthy — GSLB CRs: garage-s3-cdn
