# Phase 3: OIDC Canonicalization Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** One literal OIDC issuer (`https://pocket-id.keiretsu.top`) everywhere, auth-as-a-component for new apps, and no plaintext OIDC secrets in git.

**Architecture:** pocket-id's `APP_URL` is already `https://pocket-id.keiretsu.top` (so tokens already claim that issuer); every SecurityPolicy that spells it differently is drift. A new Kustomize Component (`clusters/common/components/oidc-protect`) stamps out the shared-client SecurityPolicy + Secret pair from `${APP}`/`${APP_HOST}`; hubble-ui converts to it as the reference consumer. code-server keeps its dedicated pocket-id client but gets the canonical issuer and its committed plaintext client secret moved into SOPS.

**Tech Stack:** Envoy Gateway SecurityPolicy (v1alpha1), Kustomize Components, Flux two-stage substitution, SOPS (PGP), flate CI.

### Task 1: The oidc-protect component

**Files:**
- Create: `clusters/common/components/oidc-protect/kustomization.yaml`
- Create: `clusters/common/components/oidc-protect/securitypolicy.yaml`
- Create: `clusters/common/components/oidc-protect/secret.yaml`

- [ ] **Step 1: Create the component**

`clusters/common/components/oidc-protect/kustomization.yaml`:

```yaml
---
apiVersion: kustomize.config.k8s.io/v1alpha1
kind: Component
resources:
  - securitypolicy.yaml
  - secret.yaml
```

`clusters/common/components/oidc-protect/securitypolicy.yaml`:

```yaml
---
apiVersion: gateway.envoyproxy.io/v1alpha1
kind: SecurityPolicy
metadata:
  name: ${APP}-oidc
spec:
  targetRefs:
    - group: gateway.networking.k8s.io
      kind: HTTPRoute
      name: ${APP}
  oidc:
    provider:
      issuer: "https://pocket-id.keiretsu.top"
    clientID: "${ENVOY_GATEWAY_OIDC_CLIENT_ID}"
    clientSecret:
      name: ${APP}-oidc-secret
    redirectURL: "https://${APP_HOST}/oauth2/callback"
    logoutPath: "/logout"
```

`clusters/common/components/oidc-protect/secret.yaml`:

```yaml
---
apiVersion: v1
kind: Secret
metadata:
  name: ${APP}-oidc-secret
type: Opaque
stringData:
  client-secret: "${ENVOY_GATEWAY_OIDC_CLIENT_SECRET}"
```

Contract (goes in the component header comment of kustomization.yaml, verbatim):

```yaml
# oidc-protect: SSO in front of an HTTPRoute via the shared envoy-gateway
# pocket-id client. Opt in from an app's app/kustomization.yaml:
#   components:
#     - ../../../components/oidc-protect        # adjust relative depth
# and set in the app's Flux ks.yaml:
#   spec.postBuild.substitute:
#     APP: <httproute name>
#     APP_HOST: <public host of the app, e.g. myapp.keiretsu.top>
# Requires the app's ks.yaml to substituteFrom common-secrets (for
# ENVOY_GATEWAY_OIDC_CLIENT_ID/SECRET) — the standard cluster wiring already
# does this. Register https://<APP_HOST>/oauth2/callback on the shared
# envoy-gateway client in pocket-id.
```
